# 🇬🇧 QoS Specification / 🇷🇺 Спецификация QoS

## Основные определения

- **SLI (Service Level Indicator)** – количественная метрика, измеряемая системой (доступность, задержка, частота ошибок).
- **SLO (Service Level Objective)** – целевое значение SLI за определённый период (например, «доступность ≥ 99.9% за 30 дней»).
- **SLA (Service Level Agreement)** – формальное соглашение с клиентом, основанное на SLO, с указанием компенсаций при нарушении.

## Поддерживаемые SLI

| SLI | Описание | Единица измерения |
|-----|----------|-------------------|
| `availability` | Доля успешных HTTP‑ответов (2xx, 3xx) от общего числа запросов | % |
| `latency_p95` | 95‑й процентиль времени ответа | мс |
| `latency_p99` | 99‑й процентиль времени ответа | мс |
| `throughput` | Количество запросов в секунду | req/s |
| `error_rate` | Доля ответов с ошибками (5xx) | % |

## Целевые SLO (пример)

| SLI | Цель | Период |
|-----|------|--------|
| `availability` | ≥ 99.9% | 30 дней |
| `latency_p95` | ≤ 200 мс | 30 дней |
| `latency_p99` | ≤ 500 мс | 30 дней |
| `error_rate` | ≤ 0.1% | 30 дней |

## Проверка SLO

Система регулярно вычисляет фактические значения SLI на основе агрегированных метрик и сравнивает их с заданными SLO. Результат отображается через GraphQL API.

## Бакеты ошибок

Для детального анализа ошибок метрики классифицируются по типам:
- **400** – ошибки клиента,
- **500** – ошибки сервера,
- **timeout** – превышение времени ожидания.

Распределение ошибок сохраняется во временных окнах для последующей визуализации.

## Проверка частоты запросов

Система отслеживает частоту входящих запросов (RPS) и сравнивает её с пороговыми значениями. При обнаружении аномальных всплесков или падений фиксируется событие.

## Пороги скорости ответа

Для задержки задаются пороговые значения (например, p95 ≤ 200 мс). При превышении порога система фиксирует нарушение SLO.

## Отображение статуса

- **ok** – SLO соблюдается.
- **warning** – SLO близок к нарушению (например, превышение порога на 10%).
- **breach** – SLO нарушен.

## Пример конфигурации SLO

```yaml
slo:
  availability:
    target: 99.9
    period: 30d
  latency_p95:
    target: 200
    period: 30d
  latency_p99:
    target: 500
    period: 30d
  error_rate:
    target: 0.1
    period: 30d
```
