# 🇬🇧 QoS Specification / 🇷🇺 Спецификация QoS

## Основные определения

- **SLI (Service Level Indicator)** — количественная метрика, измеряемая системой.
- **SLO (Service Level Objective)** — целевое значение SLI за определённый период.
- **SLA (Service Level Agreement)** — формальное соглашение с клиентом, основанное на SLO.

## Поддерживаемые SLI

| SLI | Описание | Единица измерения |
|-----|----------|-------------------|
| `availability` | Доля успешных HTTP‑ответов (2xx, 3xx) от общего числа запросов | % |
| `latency_p95` | 95‑й процентиль времени ответа | мс |
| `latency_p99` | 99‑й процентиль времени ответа | мс |
| `throughput` | Количество запросов в секунду | req/s |
| `error_rate` | Доля ответов с ошибками (5xx) | % |

## Целевые SLO (пример)

| SLI | Цель | Период |
|-----|------|--------|
| `availability` | ≥ 99.9% | 30 дней |
| `latency_p95` | ≤ 200 мс | 30 дней |
| `latency_p99` | ≤ 500 мс | 30 дней |
| `error_rate` | ≤ 0.1% | 30 дней |

## SLA

На основе SLO формируется клиентское SLA:
- Доступность ниже 99.9% → пересчёт оплаты.
- Задержка p95 выше 200 мс → эскалация.

## Сбор метрик

Метрики поступают через Kafka, агрегируются и сохраняются в PostgreSQL.
Расчёт SLI выполняется ежедневно по скользящему окну.
Результаты доступны через REST API и дашборд.
