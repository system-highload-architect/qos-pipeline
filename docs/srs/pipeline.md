# 🇬🇧 Pipeline SRS / 🇷🇺 Техническое задание на конвейер

## 🇬🇧 Overview / 🇷🇺 Обзор

The Pipeline service implements a multi‑stage data processing engine with built‑in backpressure. It consumes raw metrics from Kafka, normalises them, filters duplicates, aggregates by time windows, and writes the results to the Aggregator.
Сервис Pipeline реализует многостадийный движок обработки данных с встроенным backpressure. Он потребляет сырые метрики из Kafka, нормализует, фильтрует дубликаты, агрегирует по временным окнам и записывает результаты в Aggregator.

## 🇬🇧 Functional Requirements / 🇷🇺 Функциональные требования

- **FR‑PIPE‑001**: Конвейер должен состоять из стадий: нормализация, фильтрация, агрегация, запись.
- **FR‑PIPE‑002**: Каждая стадия должна работать в нескольких горутинах (настраиваемое количество воркеров).
- **FR‑PIPE‑003**: Между стадиями должны быть буферизированные каналы фиксированной ёмкости.
- **FR‑PIPE‑004**: При заполнении канала отправитель должен блокироваться (backpressure).
- **FR‑PIPE‑005**: Стадия фильтрации должна использовать `timedcache` для дедупликации.
- **FR‑PIPE‑006**: Стадия агрегации должна вычислять сумму, количество, минимум, максимум, p50, p95, p99.
- **FR‑PIPE‑007**: Стадия записи должна быть защищена Circuit Breaker для предотвращения каскадных сбоев.

## 🇬🇧 Non‑Functional Requirements / 🇷🇺 Нефункциональные требования

- **NFR‑PIPE‑001**: Конвейер должен обрабатывать не менее 1000 сообщений в секунду на одном воркере.
- **NFR‑PIPE‑002**: Задержка обработки одного сообщения не должна превышать 100 мс (p95).
- **NFR‑PIPE‑003**: Все стадии должны поддерживать корректное завершение при получении сигнала SIGTERM.
